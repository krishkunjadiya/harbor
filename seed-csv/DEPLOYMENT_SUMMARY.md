## Dual-ID Strategy Implemented ✓

All tables now support two ID columns:

### 1. UUID Primary Key (id)
- **Type:** UUID
- **Source:** Auto-generated by PostgreSQL
- **Purpose:** Database relationships, foreign keys
- **Example:** `3c8f3d5a-7b2e-4f6a-9d1c-5e8a9b2c1d4e`

### 2. CSV Compatibility ID ({table}_id)
- **Type:** TEXT UNIQUE
- **Source:** From CSV files
- **Purpose:** Data traceability, import consistency
- **Examples:**
  - `dep-0001` (departments)
  - `fac-0042` (faculty)
  - `crs-0015` (courses)
  - `proj-0023` (student_projects)

## Troubleshooting

### Issue: CSV Import Fails with "column does not exist"
**Solution:** Run sync-schema-with-csv.sql first to add missing columns

### Issue: Duplicate key violation on id
**Solution:** CSV already has UUID ids, let database auto-generate or use CSV values

### Issue: Array columns not importing correctly
**Solution:** Check delimiter - should be semicolon (;) for TEXT[] columns

### Issue: JSONB columns showing as text
**Solution:** Import as text, PostgreSQL will parse JSON automatically
