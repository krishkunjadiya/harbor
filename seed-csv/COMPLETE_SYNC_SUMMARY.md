## Dual-ID Strategy Implemented

All tables now have TWO ID columns:

### 1. UUID Primary Key (id)
- **Type:** UUID
- **Purpose:** Database primary key, foreign key relationships
- **Source:** Auto-generated by PostgreSQL or provided in CSV
- **Example:** `b1a1d1e1-0001-4000-8000-000000000001`

### 2. Text Compatibility ID ({table}_id)
- **Type:** TEXT UNIQUE
- **Purpose:** CSV import tracking, data traceability
- **Source:** From CSV files
- **Examples:**
  - `stu-0001` (students)
  - `rec-0042` (recruiters)
  - `bdg-0015` (Credentials)
  - `job-0023` (jobs)

## Important Notes

### Empty ID Columns in CSV
Many CSV files now have empty `id` columns. You have two options:

**Option A: Let Database Auto-Generate** (Recommended)
- Don't populate `id` columns in CSV
- PostgreSQL will auto-generate UUIDs on import
- Simpler and safer

**Option B: Use Existing UUIDs**
- Populate `id` columns with valid UUIDs
- Useful if you need specific UUIDs for relationships
- More control but requires careful management

### Foreign Key Relationships
After import, verify FK relationships work:

```sql
-- Check students link to profiles
SELECT s.student_id, s.profile_id, p.email 
FROM students s
JOIN profiles p ON s.profile_id = p.id
LIMIT 5;

-- Check jobs link to recruiters
SELECT j.job_id, j.title, r.company_name
FROM jobs j
JOIN recruiters r ON j.recruiter_id = r.profile_id
LIMIT 5;
```
